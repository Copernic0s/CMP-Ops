# Hermes Skills Map

## Useful Skills

- `frontend-design` for any Hermes UI or dashboard work.
- `github` or `github:yeet` for GitHub repository, branch, PR, and push workflows.
- `openai-docs` when the work touches OpenAI product usage or API guidance.
- `visualize` for diagrams or workflow maps when the task gets complex.

## When To Reach For Them

- Use `frontend-design` before changing visual layout, spacing, or hierarchy in the dashboard.
- Use `github:yeet` when the goal is to ship verified local changes to the remote repo.
- Use `github` when you need repository context or PR metadata.
- Use `openai-docs` only when the task is about OpenAI products or API behavior.
- Use `visualize` when a process needs a diagram instead of more prose.

## Hermes Preference

- Keep the UI operator-focused, dark, and readable.
- Prefer docs that can survive a context reset.
- Keep the repo organized around the task list and current state.
